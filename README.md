To-Do App QA Test Suite
Project Overview

This project showcases manual QA testing for a To-Do web application. I tested the core features of the app, created test cases, and documented bugs to better understand how real-world QA processes work.

Project Goal

The goal of this project was to practice hands-on QA testing by working through common user scenarios, identifying issues, and documenting them clearly.

Testing Scope

The following features were tested:

Add task
Delete task
Edit task
Mark task as complete
Input validation
Test Approach

I performed manual testing by focusing on:
Checking if features work as expected (functional testing)
Trying unusual inputs (edge cases)
Verifying how the app handles user input
Observing UI behavior
Test Coverage
10 Test Cases
4 Bug Reports
Tools Used
Manual Testing
Test Case Documentation
Bug Reporting


Skills Demonstrated
Writing clear and structured test cases
Identifying and reporting bugs
Testing application functionality
Attention to detail


What I Learned
How to create and organize test cases
How to document bugs clearly with steps and results
The importance of testing edge cases
How QA testing works in real scenarios
